What is JWT?

JWT is a bas64 encoded string consisting of a payload and signature

It's possible to encrypt it, but I haven't seen an encrypted JWT

Can be used as a stateless authentication token, but often has state attached to it

Benedikt Bauer, Beyonder - @mastacheata – DjangoCon Europe 2026

Why you might want JWT

  1. Third party identity provider gives you a JWT
  2. Save Database queries (i.e. store profile info, permissions in JWT)
    1. This comes at a price
    2. If you query the user/permissions table anyway, this might be void
  3. Mobile/Desktop/Browser apps with offline mode
    1. If you don't store the profile/permissions info in the token, a generic Token might still be cheaper

Benedikt Bauer, Beyonder - @mastacheata – DjangoCon Europe 2026

Reasons for using something else

JWTs are immutable – everything stored in there is fixed until the token expires

Revoking single tokens is impossible – same for locking users out from your application

Revoking all tokens of one user is impossible - "log out from all devices"

Benedikt Bauer, Beyonder - @mastacheata – DjangoCon Europe 2026

Tl;dr – What should I use?

  • Use JWT if imposed from outside
  • Use Session(-Cookie)-Auth for WebApps
  • (other) Token-Auth (built-in with Django) for Mobile/Desktop-Apps
